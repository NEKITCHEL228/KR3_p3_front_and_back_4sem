# Практика 17 — Отложенные Push‑уведомления (задание студентам)

## Цель практики

Сделать в PWA «напоминания», которые приходят **не сразу**, а **через заданное время** (например, через 30 секунд / 2 минуты), и добавить действие в уведомлении **«Отложить на 5 минут»**.

База проекта уже содержит:

- HTTPS запуск (нужно для SW/Push)
- skeleton под VAPID и подписку
- endpoints для тестового push
- заготовку под планировщик на сервере
- заготовку в Service Worker под action `snooze_5m`

Ваша задача — **доделать** недостающие части.

---

## 1) Быстрый старт

1. Установить зависимости:

```bash
cd server
npm i
```

2. Сделать HTTPS сертификаты (любой способ из README). Если уже сделали практики 15-16 пропускаете этот шаг (п. 3-4) и просто докидываете в свой проект заготовки из файлов этого репо.

3. Сгенерировать VAPID:

```bash
cd server
npm run vapid
```

4. Создать `server/.env` и вставить ключи.

5. Запустить:

```bash
cd server
npm run dev
```

Открыть:

- `https://localhost:3443`

---

## 2) Что уже сделано в коде

### Сервер

- `server/src/server.js`
  - `POST /api/push/subscribe` — сохранить subscription
  - `POST /api/push/test` — отправить push сразу
  - `POST /api/reminders/schedule` — **заготовка**: запланировать отправку через `delaySeconds`
  - `POST /api/reminders/snooze` — **заготовка**: «отложить на 5 минут»

### Service Worker

- `sw.js`
  - `push` — показать notification
  - `notificationclick` — заготовка под обработку `snooze_5m`

### Клиент

- `app.js` — минимальный UI, но часть действий помечена TODO.

## 4) Минимальные критерии зачёта

1. PWA работает по HTTPS и Service Worker зарегистрирован.
2. Push подписка создаётся и сохраняется на сервере.
3. Уведомление можно запланировать на N секунд (отложенная доставка работает).
4. В уведомлении есть action «Отложить на 5 минут» и он действительно перепланирует доставку.
